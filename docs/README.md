# EDA-Schema Documentation

This directory contains the canonical narrative for the EDA-Schema project: the schema description, dataset specification, quick start, API reference, and publish-ready figures. Each `.rst` file under `docs/` is included in `docs/index.rst` so the generated `docs/_build/html` site mirrors the published paper.

## Structure

- `index.rst` – landing page that assembles the major sections and links to the autogenerated API reference (`eda_schema.rst`).
- `datamodel.rst` – multimodal schema description with ER diagrams, graph/entity summaries, and metrics tables.
- `open_dataset.rst` – dataset specifications, operating points, and parameter sweeps for the OpenROAD release.
- `getting_started.rst` – quick-start tutorials, dataset exploration steps, and visualization references.
- `eda_schema.rst` – API surface generated via `autodoc`; maintains the reference sections and membership listings.

## Building locally

```bash
pip install -r requirements-dev.txt
cd docs
make html
```

The generated site appears under `docs/_build/html/index.html`. Run `make clean` before rebuilds if you encounter stale doctrees.

## Adding new content

- Update or add `.rst` files here and include them (or reference them) from `index.rst`.
- Keep diagrams/images in `docs/images/` and reference them via `.. image:: images/foo.png`.
- If you need to add graphic assets, place them under `docs/images` so Sphinx copies them into the build output.
